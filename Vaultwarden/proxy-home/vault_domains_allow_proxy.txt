# ================================
# BunkerWeb: DNSBL feeds
# ================================
# BunkerWeb's DNSBL plugin (USE_DNSBL=yes, DNSBL_LIST="bl.blocklist.de
# dnsbl.dronebl.org") checks each connecting client IP against these zones
# by issuing an A query for <reversed-IP>.<zone>. Apex match needed because
# every check produces a different subdomain (the reversed IP).
# Strictly speaking these are DNS-only (Squid doesn't see them), but kept
# here for symmetry with vault_domains_allow_dns.txt.
.dnsbl.dronebl.org
.bl.blocklist.de

# ================================
# Reverse DNS (PTR queries)
# ================================
# Linux libc does rDNS on every connect for connection logging, BunkerWeb
# client identification, etc. The reversed IP becomes a subdomain of
# in-addr.arpa (IPv4) or ip6.arpa (IPv6). DNS-only (Squid doesn't see
# PTR queries), but kept here for symmetry with vault_domains_allow_dns.txt.
.in-addr.arpa
.ip6.arpa
