# Pi-hole allowlist for the Vaultwarden VM
#
# Source of truth for what domains the Vault VM is allowed to resolve.
# Mirrors proxy-home/vault_domains_allow_proxy.txt (Squid HTTP/HTTPS
# allowlist) in a Pi-hole-compatible format.
#
# DEPLOY (three steps, all group-scoped to vaultwarden-vm):
#
# STEP 1: bulk exact-match entries via Pi-hole's "Add allowlist" URL.
#   Pi-hole web UI -> Lists -> click "Add allowlist" (green button).
#     Address: https://raw.githubusercontent.com/DiogoF-Hub/Homelab/main/Vaultwarden/vault_domains_allow_dns.txt
#     Group:   ONLY vaultwarden-vm (untick Default)
#     Comment: "vault VM allowlist (mirror of Squid)"
#   Then Tools -> Update Gravity. Pi-hole parses the bare-domain lines
#   below (everything in the "BULK" section) and adds each as an
#   exact-match allow.
#
# STEP 2: apex+subdomain entries, added manually as Allow Regex.
#   Pi-hole's allowlist URL parser does NOT accept ABP syntax (||apex^),
#   even though the blocklist URL parser does. So apex matches have to
#   go in by hand. For each line in the "APEX (manual regex)" section
#   at the bottom of this file:
#     Domain Management -> Domains -> Add a new domain
#       Type:   Allow Regex
#       Domain: <paste the (^|\.)...$ regex>
#       Group:  ONLY vaultwarden-vm (untick Default)
#       Comment: free-form (e.g. the section header from this file)
#
# STEP 3: the deny-everything-else.
#   Domain Management -> Domains -> Add a new domain
#     Type:   Deny Regex
#     Domain: .*
#     Group:  ONLY vaultwarden-vm (untick Default)
#     Comment: "vault-deny-all: catches everything not allowed above"
#
# Pi-hole evaluation order: Allow > Deny. So the Vault VM resolves only
# domains matched by step 1 OR step 2; everything else hits the .* deny
# from step 3 and returns 0.0.0.0.
#
# UPDATING (after the initial deploy):
#   - Bulk entries (step 1): edit this file in the repo, push, then in
#     Pi-hole UI run "Update Gravity" (or wait for the next scheduled
#     gravity run, default daily). Pi-hole re-fetches the URL and
#     reconciles the gravity DB.
#   - Apex regex entries (step 2): add/remove via UI by hand. The list
#     below is the source of truth, refer to it when adding new ones.
#   - When you add a new outbound destination, mirror it in
#     proxy-home/vault_domains_allow_proxy.txt as well.
#
# Future improvement: a generator that emits BOTH this file (with
# apex entries enumerated as exact subdomains, or pushed directly to
# Pi-hole's gravity DB / Allow Regex table) AND the Squid file from a
# single source. Tracked in ideas.md. Until then, two-file sync.

# APEX (manual regex, add via Domain Management -> Allow Regex)
# ================================================================
#
# These can't be ingested via the URL feature, see the deploy steps
# at the top of this file. Add each as an Allow Regex scoped to the
# vaultwarden-vm group only. Listed here so the source of truth lives
# in the repo even though Pi-hole stores the deployed copy.
#
#   (^|\.)debian\.pool\.ntp\.org$        # Debian NTP pool (0./1./2./3.)
#   (^|\.)lencr\.org$                    # Let's Encrypt OCSP + issuer chain (r10/r11/e5/e6/x1/x2)
#   (^|\.)argotunnel\.com$               # Cloudflared tunnel (region1./region2./protocol-v2./update./cfd-features./_v2-origintunneld._tcp/...)
#   (^|\.)r2\.cloudflarestorage\.com$    # Cloudflare R2 bucket-prefix subdomains
#   (^|\.)bunkerity\.com$                # BunkerWeb assets / blacklist feeds (apex + subs)
#   (^|\.)bunkerweb\.io$                 # BunkerWeb update + telemetry (apex + subs)
#   (^|\.)db-ip\.com$                    # DB-IP MMDB (apex + download.)
#   (^|\.)dnsbl\.dronebl\.org$           # BunkerWeb DNSBL plugin (DroneBL); reversed-IP becomes a subdomain
#   (^|\.)bl\.blocklist\.de$             # BunkerWeb DNSBL plugin (blocklist.de); reversed-IP becomes a subdomain
#   (^|\.)crowdsec\.net$                 # CrowdSec hub + console + community blocklist
#   (^|\.)cloudfront\.net$               # AWS CloudFront-hosted content (random subdomains)
#   (^|\.)in-addr\.arpa$                 # IPv4 reverse DNS (libc rDNS on every connect); broad but rDNS is informational only
#   (^|\.)ip6\.arpa$                     # IPv6 reverse DNS (same)
