# Pi-hole allowlist for the Vaultwarden VM
#
# This file is the source of truth for what domains the Vault VM is
# allowed to resolve. It mirrors proxy-home/vault_domains_allow_proxy.txt
# (the Squid HTTP/HTTPS allowlist) in a Pi-hole-compatible format
# (gravity / ABP syntax) so it can be consumed directly via Pi-hole's
# "Add allowlist" URL feature.
#
# DEPLOY (one-time):
#   1. Push this repo so the file is reachable at:
#        https://raw.githubusercontent.com/DiogoF-Hub/Homelab/main/Vaultwarden/vault_domains_allow_dns.txt
#   2. Pi-hole web UI -> Lists -> click "Add allowlist" (green button).
#        - Address: paste the raw.githubusercontent.com URL above
#        - Group assignment: ONLY vaultwarden-vm (untick Default)
#        - Comment: "vault VM allowlist (mirror of Squid)"
#   3. Tools -> Update Gravity (or wait for the next scheduled run).
#      Confirm the new entries land in the gravity DB by clicking the
#      list; the entry count should match the non-comment lines below.
#   4. Add ONE Deny Regex manually so anything NOT in the allowlist gets
#      blocked at DNS time:
#        Domain Management -> Domains -> Add a new domain
#        - Type:    Deny Regex
#        - Domain:  .*
#        - Group:   ONLY vaultwarden-vm (untick Default)
#        - Comment: "vault-deny-all: catches everything not allowlisted"
#
# Pi-hole evaluation order: Allow > Deny. So the Vault VM resolves only
# domains matched by an entry below; everything else hits the .* deny
# and returns 0.0.0.0.
#
# UPDATING (after the initial deploy):
#   Edit this file in the repo, push, then in Pi-hole UI run "Update
#   Gravity" (or wait for the next scheduled gravity run, default daily).
#   Pi-hole re-fetches the URL and reconciles the allow entries against
#   it. No need to re-add the list or restart anything.
#
# SYNTAX (Pi-hole / gravity):
#   exact.host.example     - exact-match allow (no subdomains)
#   ||apex.example^        - ABP-style; allow apex + ALL subdomains
#
# Comments via `#` are stripped by gravity. Inline `# blah` after an
# entry is fine; gravity just keeps the domain/ABP token.
#
# SYNC NOTE: paired by hand with proxy-home/vault_domains_allow_proxy.txt
# (Squid HTTP/HTTPS gate). Drift between the two files breaks resolution
# silently in one direction or the other (Squid lets HTTP through but
# DNS fails, or vice versa). The future single-source-of-truth
# automation that would emit both formats from one input is tracked in
# ideas.md.
